## Git Branch

Github는 다양한 기능들을 병렬적으로 다른 branch에서 개발하여 개발의 안정성을 보장해주는 Branch기능을 지원한다.

### 1. git branch

```
git branch
```
명령 프롬프트에서 Repository가 있는 경로로 이동한 후 해당 명령어를 입력하면 어떤 branch가 있는지 나온다. branch를 따로 만들지 않았으면 기본적으로 master만 있을 것이다.

<center><img width="1000" alt="2021-07-07" src="https://user-images.githubusercontent.com/53667002/124961384-5d2e6e80-e058-11eb-817c-9f3426e88e36.png"></center>


branch를 추가하려면 아래 명령어를 입력하면 된다.

```
git branch (추가할 branch)

# ex) git branch develop
```

<center><img width="1000" alt="2021-07-07 (1)" src="https://user-images.githubusercontent.com/53667002/124961526-91a22a80-e058-11eb-9c9d-4d2bc7855977.png"></center>


### 2. git checkout

git checkout으로 branch 이동이 가능하다.

```
git checkout (이동할 branch)

# ex) git checkout develop
```

<center><img width="1000" alt="2021-07-07 (3)" src="https://user-images.githubusercontent.com/53667002/124961804-e04fc480-e058-11eb-930a-d545ff49a7a9.png"></center>

branch를 이동한 상태에서 소스코드 수정, add, commit을 한 후에 log를 찍어보면 HEAD가 develop branch로 이동했고 해당 branch에서 수정된 내역이 있다는 것을 확인할 수 있다.
